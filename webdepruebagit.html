Probando comandos de Git

Subir un proyecto existente a GitHub

  • desde la web de GitHub

Creamos un nuevo repositorio en GIT Le damos nombre, descripción, seleccionamos si va a ser un proyecto público o privado si es el caso, y dejamos el check de crear README sin marcar. Le damos a crear repositorio y con esto ya tenemos el repositorio donde alojaremos nuestro proyecto.

Desde el equipo

  1. cd ruta de tu proyecto ej: cd C:\Users\oscar\OneDrive\Escritorio\DAM\ENTORNOS DE DESARROLLO
  2. git init (podemos crear un nuevo repositorio en GitHub o subirlo a un repositorio ya existente) PARA INICIAR
  3. git remote add origin ej: git remote add origin https://github.com/osscar369/Markdown.git PARA INTRODUCIR LA URL DE GIT
  4. git add –all PARA SUBIR TODO
  5. git commit -m "Commit inicial"
  6. git push -u origin master

    7. Clonar (descargarse) un proyecto de BitBucket o Github (Método manual)

    • cd (en el que vas a guardar el clon)
    • git clone + url repositorio + nombre carpeta (para clonar y descargar)

    Añadir algo a un proyecto existente

    Añadimos ficheros nuevos o hacemos cambios en ficheros existentes de nuestro proyecto…
    • git add --all
    • git commit -m "Segundo commit"
    • git push -u origin master
